Features

A change log will be added in the near future to track new and updated features

  • Route Calls During/After Work Hours
    • Choose your own physical number to direct calls to
    • Allow clients to be routed through to physical phone
    • Suited for organizations with regularly-houred office staff
  • Ensure Reliability With a Callback Schedule
    • The OnCall system will try multiple times to call the 'On Call' employee
    • If the 'On Call' employee fails to answer a set number of calls, the call will be elevated and routed to a different employee, usually a manager
    • This callback schedule is customizable and ensures that a client's call is answered to in a timely manner
  • Receive Transcription and Recording of Request Calls
    • Receive a text message with the audio transcription for quick access
    • A secure, time-sensitive link to the audio recording of the call is also included in the text. This allows users to easily listen to recordings and repeat parts of the recording that are crucial or hard to understand
    • When the 'On Call' employee is notified of an alert, the system gives them the option to listen to a text-to-speech rendition of the audio transcription. This is a limitation of Amazon's Connect system, and something for which we are looking for a workaround

AWS Cloud Computing

Below is a brief overview of the various AWS functionalities utilized by our backend services

  • Appsync/GraphQL API
    • Handles API calls made from the app and backend services
    • Allows only authorized users to access data
  • Amplify
    • Project development console with CLI/Web interface
  • Connect
    • Handles phone calls made to/from the system
    • Assigns a unique virtual phone number to each 'company'
  • Cognito
    • Allows users to securely log in to the system
    • Provides authorization when accessing data
  • DynamoDB
    • NoSQL database for storing system and user information
    • Efficient indexing schemes allow for very fast data access
  • Lambda
    • Severless functions provide functionality for backend services
  • S3
    • Stores the recordings from 'request' calls with encryption
